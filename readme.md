# English Learning App

Мобильно-адаптивное веб-приложение для изучения английских слов.

Стек: FastAPI + PostgreSQL + SPA (vanilla JS + Tailwind + Chart.js), запускается в Docker Compose.

## Возможности

- Регистрация и авторизация (JWT).
- Режим «Изучать» с двумя табами:
  - **Карточки** — активные слова в случайном порядке, с мульти-фильтрами (категории/сложность/часть речи) и кнопкой «Сбросить всё». Кнопки «Отметить прочитанным» больше нет: слова в банке появляются автоматически.
  - **Словарь** — скролл-лист всех слов в банке (Изучаем + Изучено), поиск, сортировка по активности/алфавиту/прогрессу (+ направление), клик по строке открывает модалку с деталями и формами.
- **Гибридная выдача новых слов**: в каждой категории держится до `active_slots` (по умолчанию 5) непройденных слов; в каждой категории не более `daily_new_limit` новых слов в сутки (по умолчанию 10) — лимит считается отдельно для каждой категории. Оба параметра редактируются в профиле. Как только слово становится изученным в квизе — автоматически открывается следующее, с приоритетом лёгких слов (`easy → medium → hard`).
- В «Карточках» всегда выбрана одна категория (по умолчанию — первая); рядом есть чип «🌐 Все» для выдачи сразу по всем категориям.
- В профиле есть **Опасная зона** с кнопкой сброса прогресса: требует подтверждения с 5-секундным обратным отсчётом, бэкенд-ручка `POST /auth/me/reset-progress`.
- Режим «Квиз» — английское слово и 6 вариантов перевода. Подрежимы: все слова / по категории, тумблер «Только из „Изучаем“». Пикер категорий по умолчанию прячет пустые (с нулевым банком) и показывает у каждой счётчик «Изучаем · Изучено».
- Статусы слов: **Новое** (не в банке), **Изучаем** (в процессе), **Изучено** (≥ 5 правильных ответов).
- Формы слова (времена глаголов, мн. число и т.п.) со своими счётчиками.
- Статистика: прогресс по дням, ошибки, точность, разбивка по категориям с sticky-панелью сортировки и переключателем возрастание/убывание.
- Адаптивный каркас: на десктопе — боковая навигация слева, на мобильных — нижний tab-bar.
- **Озвучка английских слов** (Piper/Kokoro на CPU): кнопка «🔊» у слова на карточке, в деталях и в квизе. Аудио генерируется один раз и кешируется в БД (`word_audio`). В профиле переключается «Тренировка на слух» — английское слово скрывается под блюром до прослушивания.

## Быстрый запуск

```bash
cp .env.example .env
make up
```

После старта:

- Фронт: http://localhost:8080
- API:  http://localhost:8080/api/v1 (проксируется на backend)
- Swagger: http://localhost:8000/docs (при выставленном порте backend)

## Безопасность

Чек-лист перед выкатом на VPS:

1. `cp .env.example .env` и заменить:
   - `POSTGRES_PASSWORD` и пароль внутри `DATABASE_URL` — строго одинаковые;
   - `JWT_SECRET` — результат `make gen-secret` (64+ случайных символов);
   - `CORS_ORIGINS` — только реальные origin'ы фронта (указывать явно, `*` запрещён).
2. Задать `ADMIN_EMAIL` в `.env`. При старте контейнера (`make up` / `make restart`) автоматически выполняется `python -m app.scripts.create_admin --auto`: если пользователь с таким email уже есть в БД — ему выдаётся `is_admin=True`, иначе шаг тихо пропускается. Так что достаточно зарегистрироваться через фронт и перезапустить backend. Можно и вручную в любой момент: `make create-admin` (возьмёт email из `.env`) или `make create-admin email=<другой>`.
3. Firewall VPS: оставить снаружи только `22/tcp` (SSH) и `80/tcp` (фронт/API). Порт backend`:8000` внутри compose-сети и наружу не торчит.
4. Когда появится домен — раскомментировать TLS-блок в `frontend/nginx.conf`, положить сертификаты в `frontend/certs/`, включить HSTS.

Что уже защищено «из коробки»:

- **Rate limit** в nginx (`auth` — 5 req/min, общий API — 30 req/s) и в FastAPI через slowapi (login/register/reset/refill — 5/мин; quiz — 60/мин; audio — 30/мин).
- **Антибрутфорс логина**: 10 подряд неудач → блокировка аккаунта на 15 минут.
- **Unified 401** — ответ на неверный email и неверный пароль неотличим.
- **Argon2** для новых паролей (старые bcrypt-хеши мигрируют при логине). Пароль: 8+ символов, буква + цифра обязательно.
- **JWT** с `jti` и `iat`, время жизни 2 часа по умолчанию.
- **Destructive-ручки** (`/auth/me/reset-progress`, `/auth/me/refill-words`) требуют повторный ввод пароля.
- **Security-заголовки**: `Content-Security-Policy`, `X-Frame-Options: DENY`, `X-Content-Type-Options: nosniff`, `Referrer-Policy`, `Permissions-Policy`, `server_tokens off`.
- **Docker**: backend бежит под non-root uid 1001, `no-new-privileges`, лимиты `mem_limit` и `pids_limit`, healthcheck у всех сервисов, ротация json-file-логов.
- **Аудит** пишется в таблицу `audit_log` (логины, ресет прогресса, админ-действия). Смотреть: `make audit`.
- **Индексы** под регистронезависимый поиск по словам и под статистику квизов.
- **Пул БД** ограничен: `pool_size=10`, `max_overflow=10` — не даст одному всплеску положить Postgres.

## Структура

```
backend/
  app/         FastAPI + SQLAlchemy + Alembic
  words/       JSON-файлы со словами по категориям (по одному файлу на slug)
frontend/      SPA (nginx раздаёт статику, проксирует /api)
```

## Слова и категории

- 25 компактных категорий (фрукты, овощи, животные, птицы, цвета, ... базовые глаголы).
  Список живёт в `backend/app/seeds/categories.py`.
- Для каждой категории есть JSON в `backend/words/<slug>.json` с 20 ключевыми словами.
- При `make up` автоматически запускается два сида:
  1. `app.seeds.categories` — upsert категорий по slug.
  2. `app.seeds.words` — читает JSON-ы, делает **upsert** слов (по паре `category_id`+`english`)
     и форм (по `form_type`). Если JSON для существующей категории отсутствует —
     сид создаёт пустой шаблон.
- Чтобы добавить/отредактировать слово — просто правьте JSON и перезапускайте `make seed-words`.

### Формат JSON слова

```json
{
  "english": "apple",
  "russian": "яблоко",
  "description": "Круглый сочный плод",
  "transcription_ipa": "ˈæp.əl",
  "transcription_ru": "эпл",
  "difficulty": "easy",
  "part_of_speech": "noun",
  "forms": [
    {"form_type": "plural", "english": "apples", "russian": "яблоки",
     "transcription_ipa": "ˈæp.əlz", "transcription_ru": "эплз"}
  ]
}
```

- `difficulty` — `easy | medium | hard`
- `part_of_speech` — `noun | verb | adjective | adverb | pronoun | preposition | conjunction | interjection | phrase`
- `form_type` — `base | past_simple | past_participle | present_participle | third_person | plural | comparative | superlative`

## Озвучка (TTS)

Поддерживаются два движка, работающих на CPU. Переключение — через `.env`:

```
TTS_ENGINE=piper            # piper | kokoro
TTS_VOICE_PIPER=en_US-amy-medium
TTS_VOICE_KOKORO=af_heart
TTS_MODELS_DIR=/app/models/tts
```

- **Piper** — быстрый ONNX-синтез (~20–100 мс на слово), модели ~60 MB. Рекомендуемый вариант.
- **Kokoro** — нейро-TTS, качество выше, но требует `torch` (~700 MB образа) и заметно медленнее. Ставится вместе с Piper для тестов.

Модели подтягиваются автоматически при первом запуске и складываются в том `tts_models`.

Аудио-файлы кешируются в Postgres (`word_audio`, BYTEA). Ключ кеша — `(word_id, engine, voice)`, так что переключение движка/голоса не затирает старые записи.

- Очистить кеш: `make tts-clear`
- После смены `TTS_ENGINE` в `.env`: `make restart`

В профиле пользователя есть чекбокс «Тренировка на слух» — английское слово скрывается под блюром и проявляется либо по клику на слово, либо после прослушивания через кнопку динамика.
